# Context-Aware Smart Food Coach 🥑

## Overview
The Context-Aware Smart Food Coach is an intelligent web application designed for the **Health & Wellness** vertical. It acts as a customized AI assistant that analyzes a user's health goals, recent eating habits, time of day, and current mood to provide hyper-personalized meal recommendations, behavioral nudges, and smart food swaps. 

## Approach & Logic
Instead of relying solely on general AI prompting, this application features a **Smart Logic Layer** that runs _before_ passing details to the AI model. 

1. **Rule-Based Engine (The Smart Logic Layer)**:
   - Evaluates the user's input (Time + Last Meal) to calculate an initial **Health Score** base metrics. 
   - Generates a **Behavior Insight**. For example: If the user ate pizza at 11 PM, the engine detects this as "High Carb / Late Night Eating".
2. **AI Layer (Google Gemini)**: 
   - The insights generated by the Rule-Based layer are securely sent backwards to **Google Gemini (Vertex AI)** using a detailed prompt. 
   - Gemini calculates the ideal next meal to counteract or support the previous meal, explains _why_ this works to fit their goal, assigns the final Health Score, and gives a practical "Smart Swap".

## How the Solution Works
1. A user visits the highly-responsive, clean UI on the frontend.
2. They input their physical goals, describe their last meal, drop in the time of day, and (optionally) log their mood.
3. The frontend passes a secure JSON package to the Node.js / Express backend.
4. The backend processes the variables through the rule engine, builds the context prompt, and bridges to the Gemini AI model.
5. The model outputs a strict JSON payload which the backend feeds exactly to the UI, dynamically changing gradient colors and populating smart cards to display the data beautifully. 

## Assumptions Made
- Users are providing truthful text descriptions of their meals.
- This app acts purely as a "coach/guide" and not as medical/dietary prescription.
- A Node.js environment is standard and easily supported for server hosting. 
- You possess an active Google Gemini API Key.

## 🚀 How to Run Locally

1. Clone the repository and navigate into it.
2. Run `npm install` to install dependencies.
3. Create a `.env` file in the root directory and add your API key:
   ```
   GEMINI_API_KEY="your_google_gemini_api_key_here"
   ```
4. Run the server using `npm start` (or `node server.js`).
5. Open your browser and navigate to `http://localhost:3000`.

1. **Enable the Cloud Run & Build APIs**:
   Go to the Google Cloud Console and enable the `Cloud Run API` and `Cloud Build API` for your project.

2. **Initialize gcloud CLI (if not done yet)**:
   ```bash
   gcloud init
   ```
   Select your project where your $5 free credits are active.

3. **Deploy directly from source**:
   Navigate to the repository folder on your machine where `server.js` and `package.json` are located, and run the following command to securely build and deploy your app. (We bypass Docker setup because Cloud Run natively understands Node.js code).
   ```bash
   gcloud run deploy smart-food-coach \
     --source . \
     --platform managed \
     --region us-central1 \
     --allow-unauthenticated \
     --set-env-vars="GEMINI_API_KEY=your_real_gemini_api_key_here"
   ```

4. **Follow Prompts**:
   - When asked if you want to continue, hit `Y`.
   - Google Cloud Build will automatically containerize the Node.js application and deploy it.
   
5. **Access Your Live App**:
   Once the process completes (usually 2-3 mins), it will output a **Service URL** (e.g. `https://smart-food-coach-xxxxx-uc.a.run.app`).
